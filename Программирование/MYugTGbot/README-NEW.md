# MYugTGbot - Упрощенный Telegram Bot

## 📝 Описание

Telegram бот на **NestJS + Telegraf** для работы с заказами, отгрузками и журналом расходов.  
Используется **одна база данных ITM (Firebird)** вместо двух.

## ✅ Реализованные модули

### 1. 👥 Users (Пользователи)
- ✅ Регистрация и управление пользователями
- ✅ Роли и права доступа (Guest, Client, Agent, Admin, etc.)
- ✅ Профили пользователей
- ✅ **База данных**: ITM (TODO: перенести tg_users в ITM или использовать PostgreSQL)

### 2. 📦 Orders (Заказы) - В РАЗРАБОТКЕ
- 🔨 Просмотр заказов
- 🔨 Поиск заказов (по ID, дате, клиенту, материалу)
- 🔨 Упакованные заказы
- 🔨 Мои заказы (для менеджеров)
- **База данных**: ITM

### 3. 🚚 Shipments (Отгрузки) - В РАЗРАБОТКЕ
- 🔨 Отгрузки профиля
- 🔨 Отгрузки фасадов
- **База данных**: ITM

### 4. 📊 Cash Transactions (Журнал расходов) - В РАЗРАБОТКЕ
- 🔨 Просмотр кассовых операций
- 🔨 Добавление записей
- 🔨 Фильтры по дате/категории
- 🔨 Отчеты (за день, неделю, месяц)
- **База данных**: ITM

### 5. 💰 Money (Капуста) - В РАЗРАБОТКЕ
- 🔨 Баланс кассы
- **База данных**: ITM

## 🚀 Быстрый старт

### 1. Установка зависимостей
```bash
npm install
```

### 2. Настройка окружения
Создайте файл `.env`:
```env
# Telegram Bot
TELEGRAM_BOT_TOKEN=your_bot_token_here

# Firebird Database - ITM (единственная база)
ITM_DB_HOST=localhost
ITM_DB_PORT=3050
ITM_DB_DATABASE=/path/to/itm.fdb
ITM_DB_USER=SYSDBA
ITM_DB_PASSWORD=masterkey

# Application
NODE_ENV=development
PORT=3000
```

### 3. Запуск в режиме разработки
```bash
npm run start:dev
```

### 4. Сборка проекта
```bash
npm run build
```

### 5. Запуск в production
```bash
npm run start:prod
```

## 🗄 База данных

### Основная БД: ITM (Firebird)
Все данные хранятся в **одной** базе данных ITM.

**Таблицы**:
- `ORDERS` - заказы
- `JOURNAL_CASHFLOW` - кассовые операции
- `JOURNAL_UPACK` - упакованные заказы
- `tg_users` - пользователи Telegram (TODO: перенести в ITM)
- И другие...
