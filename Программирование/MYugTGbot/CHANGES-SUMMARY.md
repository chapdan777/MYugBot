## 💡 Рекомендации

### База данных tg_users
Таблица `tg_users` сейчас в запросах, но в flows.json она была в CUBIC.  
**Варианты**:
1. ✅ **Рекомендуется**: Создать таблицу в ITM
2. Использовать PostgreSQL для Telegram данных
3. Мигрировать данные из CUBIC в ITM

### Inline Keyboards
В flows.json использовались обычные клавиатуры. В новом проекте лучше использовать **только inline**:
- Лучший UX
- Не занимают место
- Можно обновлять сообщения
- Callback queries

### Код из flows.json
Много полезного кода в flows.json, который нужно портировать:
1. **Утилиты форматирования** (formatDate, formatMonetary)
2. **SQL запросы** (для Orders, Cash, Shipments)
3. **Логика поиска** заказов
4. **Генераторы сообщений**
